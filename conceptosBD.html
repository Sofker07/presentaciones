Funciones de Bases de Datos

between

La cláusula BETWEEN se utiliza para filtrar filas en una consulta que están dentro de un rango específico de valores. Normalmente se usa en combinación con los operadores lógicos AND o NOT para definir un rango

Por ejemplo, si tienes una tabla de empleados y deseas encontrar todos los empleados cuyos salarios estén entre $30,000 y $40,000, podrías usar la cláusula BETWEEN de la siguiente manera:


                            SELECT * FROM empleados WHERE salario BETWEEN 30000 AND 40000;

DISTINCT

La palabra clave DISTINCT se utiliza en una consulta SELECT para eliminar las filas duplicadas de los resultados. Cuando se usa, solo se mostrará una fila para cada valor único en la columna especificada

Por ejemplo, si tienes una tabla de clientes y deseas obtener una lista de todos los nombres de los clientes sin duplicados, puedes hacerlo de esta manera:


                            SELECT DISTINCT nombre FROM clientes;

EXTRACT

EXTRACT es una función que se utiliza para extraer componentes de una fecha o una marca de tiempo (timestamp). Puede usarse para obtener partes específicas de una fecha, como el año, el mes, el día, la hora, el minuto, el segundo, etc. Su uso varía según el sistema de gestión de bases de datos (DBMS) que estés utilizando

Aquí hay un ejemplo de cómo podrías usarlo para extraer el año de una fecha:


                            SELECT EXTRACT(YEAR FROM fecha) FROM eventos;

INNER

INNER es una parte de la cláusula JOIN que se utiliza para combinar dos o más tablas en una consulta SQL. Un INNER JOIN devuelve solo las filas que tienen coincidencias en ambas tablas. En otras palabras, solo se incluyen las filas en las que hay valores correspondientes en ambas tablas

Por ejemplo, si tienes una tabla de clientes y una tabla de pedidos, puedes usar un INNER JOIN para combinarlas y obtener información sobre los clientes que han realizado pedidos:


                            SELECT clientes.nombre, pedidos.producto
                            FROM clientes
                            INNER JOIN pedidos ON clientes.id = pedidos.cliente_id;
